## Relational Database Design (PK/  FK)
- Relational Database stores data in tables and connects those tables using relationships.
ex: department -> employees
- One department can have many employees. (ONE TO MANY RELATIONSHIP)

 ##  create employees table

 CREATE TABLE employees(
    id serial primary key,
    name varchar(50) not null,
    email varchar(100) not null,
    salary int,
    dept_id int
);

- insert into employees table

## create departments table
CREATE TABLE departments(
    id serial primary key,
    department_name varchar(50)
);

Foreign key
------------------
Foreign key prevents invalid data.
Ex: INSERT INTO employees(name, department_id)
VALUES('rahul', 10)

Error: insert or update violates Foreign key constraint
- Because department 10 does not exist.

JOINS
------
SELF JOIN:  When a Table joins with itself
- employees have managers
- categories have parent categories
- users refer other users
- In self join we usually use LEFT JOIN because some rows (like top managers) may not have a referece value. LEFT JOIN ensures all rows from the main table are returned even if there is no matching record.

 CASE WHEN -- (SQL if-Else Logic)
 CASE WHEN works like if-else statements in programming.

 CASE -- Start conditions Logic
 WHEN -- conditions
 THEN -- Result if condition true
 ELSE -- default results
 END -- End logic

SELECT name, salary,
CASE
    WHEN salary >= 60000 THEN 'High Salary'
    WHEN salary >= 40000 THEN 'Medium Salary'
    ELSE 'Low Salary'
END AS salary_category
FROM employees;

Foreign Key With ON DELETE
- you can control what happens when the parent row is deleted.

CASCADE: if department is deleted -> employees also deleted.
